Получение GPS данных с трекеров

  • Отправка “GPS” данных.

Отправка “GPS” данных.

POST /bingps?imei=:imei

Запрос

imei
Обязательный string - IMEI GSM модуля трекера. По данному полю идентифицируется трекер. Значение может представлять собой или 15-ти разрядное число, или значение число-код, где код - дополнительное числовое поле от 1 до 30000.
dataid
Необязательный integer - Числовое значение, увеличивающееся на 1 при кажной новой отправке данных на сервер.
hwid
Необязательный string - Идентификатор устройства.
swid
Необязательный string - Идентификатор версии прошивки.
csq
Необязательный string - Уровень сигнала GSM-сети, определяемый формулой (csq * 2 - 113) дБ.
vin
Необязательный string - Напряжение основного питания в миливольтах.
vout
Необязательный string - Напряжение резевного питания в миливольтах.

Тело запроса представляет собой бинарные данные. Заголовок Content-Type подразумевается установленным в application/octet-stream.

Тело запроса состоит из одного или нескольких блоков. Обычно размер каждого блока 32 байта. Трекер должен ограничивать размер общих передаваемых данных, обычно это около 16 килобайт.

Последние 2 байта тела запроса содержат циклическую контрольную сумму CRC16-CCITT : Poly=0x1021, Init=0, Revert=false, XorOut=0x0000, Check: CRC(“123456789”)=0x31C3.

Каждый блок данных имеет заголовок, равный 0xFF, за которым следует идентификатор типа данных блока, и далее 30 байт тела блока. Более подробно формат блоков данных описан в разделе Получение “сырых” данных.

Ответ

В случае успешного приема данных сервер возвращает BINGPS: OK. Сетвер проверяет только контрольную сумму, и если она совпала, то возвращается успешный ответ.

Status: 200 OK
Content-Type: application/octet-stream
Content-length: 12
BINGPS: OK

Если контрольная сумма данных не совпала, то будет возвращен ответ:

Status: 200 OK
Content-Type: application/octet-stream
Content-length: 18
BINGPS: CRCERROR
