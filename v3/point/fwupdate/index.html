Обновление внутреннего программного обеспечения трекеров

  • Запрос версии актуальной прошивки:

Поддерживается дистанционное обновление программного обеспечения. Если на какой-либо запрос трекера сервер, перед обычным ответом, отвечает строкой FWUPDATE, или если на номер карточки трекера была отправлена SMS-команда fwupdate, то трекер начинает процедуру обновления внутреннего программного обеспечения.

Запрос версии актуальной прошивки:

GET /firmware?imei=123456789012345&cmd=check&hwid=3041

Ответ

Status: 200 OK
Content-Type: application/octet-stream
SWID: 3060

Где значение после SWID: - это версия (в 16-тиричном виде) последей доступной прошивки для запрошенного hwid.

Если прошивки для запрошенного hwid недоступны, то сервер отвечает:

Status: 200 OK
Content-Type: application/octet-stream
NOT FOUND

Система, если видит необходимость обновления (версия SWID больше чем та что в нем установлена), то система запрашивает прошивку:

GET /firmware?imei=123456789012345&cmd=get&hwid=3031

Ответ

Status: 200 OK
Content-Type: application/octet-stream
SWID:3060
LENGTH:xxxx
LINE0000:000102030405060708090A0B0C0D0E0F101112131415161718191A1B1C1D1E1F
LINE0001:000102030405060708090A0B0C0D0E0F101112131415161718191A1B1C1D1E1F
LINE0002:000102030405060708090A0B0C0D0E0F101112131415161718191A1B1C1D1E1F
…
LINExxxx:000102030405060708090A0B0C0D0E0F10111213141516
CRC:xxxx
ENDDATA

где:

SWID:xxxx
версия в 16-тиричном виде.
LENGTH:xxxx
размер образа в 16-тиричном виде строки LINExxxx:, где xxxx – порядковый номер строки в 16-тиричном виде. В каждой строке LINE передается 32 байта образа представленных в 16-тиричной системе. (последняя строка может содержать менее 32-х байт образа).
CRC:xxxx
значение циклической контрольной суммы CRC16-CCITT (в 16-тиричном виде).

Завершается ответ строкой ENDDATA.
